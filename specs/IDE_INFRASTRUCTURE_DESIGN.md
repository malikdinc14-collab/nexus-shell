# Technical Design: Nexus IDE Infrastructure (V1)

**Status**: Draft for Implementation
**Author**: The Board
**Objective**: Establish the core inter-pane communication and the "Multi-View" tab framework.

---

## 1. Parallax Event Bus (`px-bus`)

### Architecture
- **Daemon**: A Python-based Unix Socket server.
- **Socket Location**: `/tmp/parallax_$USER/bus_$SESSION.sock`.
- **Mode**: Non-blocking broadcast. Every message sent to the socket is mirrored to all active subscribers.

### Protocol (JSON Envelope)
Every message must follow this schema:
```json
{
  "id": "uuid-v4",
  "timestamp": "ISO8601",
  "source": {
    "pane_id": "editor",
    "pid": 1234
  },
  "type": "NXS_EVENT_TYPE",
  "payload": { ... }
}
```

### Core CLI Interface
- `px-event pub <type> '<json_payload>'`
- `px-event sub <type> [--on-message <callback_script>]`

## 2. Generic Tab Module (`modules/tabs`)

### The State Model (`tabs.json`)
The module maintains a global registry of all viewports and their content:
```json
{
  "viewports": {
    "terminal": {
      "active_tab_id": "tab_zsh_1",
      "tabs": [
        { "id": "tab_zsh_1", "title": "zsh", "type": "shell", "pane_id": 5, "dirty": false },
        { "id": "tab_logs_1", "title": "logs", "type": "tail", "pane_id": 9, "dirty": true }
      ]
    }
  }
}
```

### Process Swapping (The "Invisible Hand")
To keep processes alive without visual clutter:
1.  **Background Window**: Create a hidden tmux window named `_nexus_background`.
2.  **The Swap**: 
    - When a tab is switched, the current active pane in the viewport is `swap-pane` with the target pane from the `_nexus_background` window.
    - **Result**: Instant switching with 100% state persistence (scrollback, running processes).

### UI Rendering
The `pane-border-format` will be dynamically generated by `tabs/renderer.py`:
- Use `#[bg=...,fg=...]` tmux format strings.
- **Active Segment**: `#[bg=cyan,fg=black,bold] tab_title #[default]`
- **Unsaved Icon**: Detect `"dirty": true` in the state and append `●`.

### Integration API (Example for Neovim)
Components notify the Tab Module via the Event Bus:
1. Neovim saves file → `px-event pub NXS_BUFFER_SAVED '{"file": "init.lua", "dirty": false}'`.
2. Tab Module Subscriber → Updates `tabs.json` -> Triggers `tmux refresh-client` to redraw border.

## 3. Key Interactions
- **Global Next/Prev**: `Alt + [` and `Alt + ]` trigger `tabs/manager.py next --viewport <focused_pane>`.
- **Global New**: `Alt + =` triggers context-aware creation (e.g., `tmux send-keys` for a new shell).
