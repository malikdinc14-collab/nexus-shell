# DSN-003: Dynamic Visualisation Panes

## Architecture Overview
The system utilizes a "Surface" mechanism to reconfigure the TUI layout on the fly, allowing for specialized monitoring views.

## 1. Monitoring Windows
- **Window 2 (Monitor)**: A new dedicated window for high-density telemetry.
- **Layout**:
    - **Top (20%)**: Custom Intelligence Dashboard (Parallax).
    - **Bottom Left (40%)**: `mactop` or system metrics.
    - **Bottom Right (40%)**: Agent Trace logs (`tail -f`).

## 2. Dynamic Pane Actions
- `px-pop <command>`: Opens a temporary pane (bottom split) with the given command, which auto-closes on completion.
- `px-switch-view <view_id>`: Reconfigures the center pane to show a specific dashboard (e.g., 'metrics', 'logs', 'editor').

## 3. Visualisation Dashboard
- A new Parallax surface `intel:monitor` will be created to aggregate data from:
    - `/tmp/px-agent-trace.log`
    - `/tmp/px-resource-guard.log`
    - `mactop` headless output.
