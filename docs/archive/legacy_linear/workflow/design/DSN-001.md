# DSN-001: Multi-Model Architecture & Rich Media TUI

## Architecture Overview
The system uses a tiered intelligence approach coupled with a custom rendering pipeline for the Nexus-Shell interface.

## 1. Intelligence Tiers
- **Tier 1 (The Scout)**: Qwen2.5-Coder-0.5B or AgentCPM (Port 8081). Handles low-latency tasks and pre-screening.
- **Tier 2 (The Expert)**: Qwen3-Coder-30B (Port 8080). Handles complex reasoning and code emission.
- **Tier 3 (Memory/OS)**: Letta Service (Port 8083). Manages persistent agent state.

## 2. RLM (Recursive Language Model) Logic
- **Decomposition**: Input is processed by the Expert to identify key sections.
- **Recursion**: Scout is dispatched to analyze sub-sections (depth=1).
- **Synthesis**: Findings are merged back to the primary context.
- **Provenance**: Logic tree is exported as a Mermaid diagram.

## 3. Visual Rendering Pipeline
- **Script**: `nexus-view.sh` acts as a wrapper around `glow`.
- **Mermaid**: Detected blocks are rendered via `mermaid-cli` to `/tmp/nexus/render.png`.
- **iTerm2 Protocol**: Images are displayed using `imgcat` escape sequences.

## 4. Hardware Management
- **Monitor**: `mactop` provides real-time JSON metrics.
- **Dashboard**: Parallax Brains pillar displays telemetry and active model map.
