# DSN-002: Live Agent Orchestration

## Overview
Transform the agent from a static side-chat into a "Ghost Operator" that executes commands and edits code live in the workspace, controlled via the global command prompt.

## 1. Interaction Model
- **Input**: Use the standard Nexus `Ctrl+\` (Global Dispatcher). Commands starting with `!` or non-standard commands are routed to the Agent Bridge.
- **Output**:
    - **Live Execution**: Commands appear in the interactive terminal (Pane 3.0).
    - **Live Editing**: Code changes appear in the Neovim buffer (Pane 2) via RPC.
    - **Trace Log**: Internal reasoning streams into the Trace Pane (Pane 3.1).

## 2. Configuration Settings
- `PX_AGENT_AUTO_EXEC`: [true/false] Immediate command execution vs. staging.
- `PX_AGENT_LIVE_EDIT`: [overwrite/diff] Direct buffer modification vs. side-by-side comparison.
- `PX_AGENT_TRACE_MODE`: [raw/clean] Token-by-token stream vs. high-level status updates.

## 3. New Commands
- `:kill`: Immediately terminates all active agent processes.
- `:down`: Unloads all local models and shuts down serving backends (MLX/Ollama/Letta).

## 4. Components
- **Bridge (`px-bridge-agent`)**: Intercepts model output and dispatches it to TMUX (keys) and Neovim (RPC).
- **Dispatcher Update**: Routes `:kill`, `:down`, and agent queries.
- **Layout Update**: Splits Terminal pane into 3.0 (Interactive) and 3.1 (Trace).
