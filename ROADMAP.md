# Nexus Shell: The Path to Agentic Sovereignty

This roadmap outlines the evolution of Nexus Shell from a terminal multiplexer into a high-performance, human-gated AI development factory.

## 🟢 Phase 1-3: Foundations (Baselined)
- [x] **UI & Session Layer**: Guarded session handling, recursive process cleanup, and YAML-driven menus.
- [x] **Engine Layer**: Theme engine, task registry, and dynamic RPC-driven searching (fd/ripgrep).
- [x] **Domain Configurations**: 34+ module manifests and 15+ specialized compositions.

## 🟡 Phase 4: IDE Parity (Desktop Killer)
- [x] **Multi-Window Support**: Native Tmux window orchestration (Slots 1-10) with branch-aware persistence.
- [ ] **The Merge Conflict Matrix**: Auto-detection of Git conflicts with instant hot-swapping to a 3-way split review layout.
- [ ] **Headless DAP (Debug Adapter Protocol)**: Decoupled debug REPLs orchestrated by Nexus, keeping the editor buffer clean while execution lives in specialized panes.
- [ ] **Global Jump-to-Definition**: Cross-pane LSP intelligence that allows clicking/hitting keys on a stack trace to instantly navigate the editor in a different pane.

## 🚀 Phase 5: AI Sovereignty & Background Services
- [x] **Daemon Manager**: Invisible background orchestration for LSPs, DAPs, and AI backends.
- [x] **Pi Frontend Integration**: Lightweight, conversational TUI bridge to background agent engines.
- [ ] **Agent Zero: Headless Migration**: Complete decoupling of the reasoning engine from the Gradio UI.
- [/] **Agent Follow Mode ("Ghost Driving")**: Real-time editor/shell orchestration where the agent physically demonstrates changes in the visible UI.
- [ ] **Dockerized Execution Sandbox**: Strict containerization for all agent tool executions to support safe, long-running scripts.
- [ ] **Pervasive Context Pumping**: Real-time telemetry streaming (active file, cursor position, build errors) into the agent's memory.
- [ ] **Semantic RAG**: Local vector embedding (LanceDB) for instant, project-wide technical answers.

## 🛡️ Phase 6: Gated Agent Protocol (GAP) Integration
- [/] **The GAP Engine**: Port the Gated Agent Protocol logic natively into Nexus Shell.
- [ ] **Scribe Subsystem**: Automated generation of `idea.md` and `design.md` proposals based on natural language intent.
- [ ] **Gate Enforcement**: Strict human-in-the-loop approval gates for all destructive agent actions.
- [ ] **Worktree Swarms**: Automated provisioning of ephemeral Git worktrees for parallel, concurrent feature development.
